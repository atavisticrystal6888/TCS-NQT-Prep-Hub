1-2
Quantitative Aptitude
Master key formulas, learn shortcuts. Practice 50+ problems on percentages, ratio, speed-distance, SI/CI, profit-loss.
3
Logical Reasoning
Pattern recognition, coding-decoding, number series, blood relations, syllogisms. Practice 30+ questions.
4
Verbal Ability
Grammar rules (articles, tenses, subject-verb), vocabulary (synonyms, antonyms), RC practice. Learn 50 new words.
5
Programming Logic + DBMS
Pseudocode tracing, output prediction, OOP concepts, SQL queries, normalization basics.
6
Coding Practice + CS Fundamentals
Solve 10-15 easy-medium problems in C/Java/Python. Review OS scheduling, network layers.
7
Full Mock Test + Revision
Take a full-length mock. Review all wrong answers. Revise formula sheet and key facts.

⏱️ Time Management Tips

  • No negative marking — attempt ALL questions, never leave blanks
  • Numerical: ~1.5 min/question. Skip if stuck beyond 2 min, return later
  • Verbal: Read questions first, then scan the passage for answers
  • Reasoning: Draw diagrams for seating & blood relations. Use elimination
  • Programming: Trace code line-by-line mentally, check edge cases
  • Coding: Think approach for 2 min before writing. Test with sample input
  • Section order: Start with your strongest section for confidence
  • Flag & return: Mark tough questions and come back in remaining time

💡 Common Mistakes to Avoid

  • Don't spend too long on a single question — move on and return
  • Don't skip questions — no negative marking means always attempt
  • Don't ignore verbal section — it's high-scoring with practice
  • Don't memorize code — understand logic and trace step by step
  • Don't panic if coding section is hard — focus on partial marks
  • Don't forget to check unit conversions (km/h ↔ m/s, hours ↔ min)
  • Don't rely solely on shortcuts — understand the concept behind them
  • Don't ignore the last 10 minutes — review flagged questions

🧠 Section-wise Pro Tips

🔢 Quantitative

  • Learn tables up to 30, squares up to 25, cubes up to 15
  • Use approximation for complex calculations
  • For percentage questions, use base value method
  • Ratio problems: assume values to simplify

🧩 Reasoning

  • In series questions, check differences then ratios
  • For coding-decoding: map A=1, Z=26 immediately
  • Blood relations: draw family tree diagrams
  • Syllogisms: use Venn diagrams, not assumptions

📝 Verbal

  • Read 1 English article daily to improve comprehension
  • Focus on common confusables: affect/effect, its/it's
  • Para jumbles: find the intro sentence first (general → specific)
  • Cloze test: read full passage before filling blanks

💻 Programming

  • For pseudocode: track variables on paper step by step
  • Know Big-O of common algorithms and operations
  • Understand recursion by drawing call stacks
  • Practice output prediction for loops and conditionals
